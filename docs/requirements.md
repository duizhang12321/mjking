# 需求澄清文档

请按下述结构逐项补充/修改，文档将作为实现的唯一事实来源。

## 一、背景与目标
- 背景：开发一个辅助线下麻将的小程序
- 目标：为线下麻将玩家提供一个辅助工具，功能包括自定义/预置麻将规则并使用规则算分，开房间记账并自动关联记分规则
- 范围边界（不做什么）：

## 二、用户与场景
- 用户角色：线下麻将玩家
- 使用场景：
  - 玩家可以开房间记录线下麻将积分
  - 房主创建房间后通过“小程序分享”邀请其他玩家加入；不支持手动添加玩家与通过房间ID加入
  - 房间支持玩家手动输入每局积分或通过上传牌型图片进行 AI 记分（与所选规则关联）
  - 规则管理支持查看预置规则（川麻模版）、新建自定义规则（仅新建/删除）
- 成功标准（验收指标/容忍度）：

## 三、规则模版（AI 算分依据）
- 存储格式：Markdown（人类可读，AI 直接消费）
- 必需分节：
  - 桌面（`players`、`handSize`、`tiles.totalTiles/是否含风/箭/花`）
  - 庄家（`continuousDealer`、`dealerBonus.selfDrawAddBase/winAddBase`）
  - 结算（`payMode`：一人付/三家付、`basePoint`、`cap`）
  - 抓鸟（`enabled`、`count`、`hitBonus`）
  - 违例检查（查大叫/查花猪）
  - 番型（基础番、杠类、额外加成；示例：平胡、自摸、对对胡、清一色、明杠、暗杠、绝张）
- 预置规则：川麻·血战到底（已内置并可查看）
- 自定义规则：输入“名称 + 自然语言描述”，调用 LLM 渲染 Markdown；保存前执行规范校验（必需分节存在）

## 四、AI 服务与架构选择
- 上传图片并计算积分通过调用AI服务实现，目前使用火山引擎的API

## 五、登录与加入
- 登录：首次进入必须微信头像/昵称授权（本地认证，不依赖后端）
- 加入：仅支持通过房间分享卡片进入后自动加入；不支持手动添加或输入房间ID加入

## 六、校验与错误处理
- 规则保存校验：
  - 结构完整性：玩家数/手牌数/番型数组存在
  - Markdown 分节：桌面/庄家/结算/抓鸟/违例检查/番型
- LLM 渲染失败回退：使用本地解析生成规范 Markdown
