# 需求澄清文档

请按下述结构逐项补充/修改，文档将作为实现的唯一事实来源。

## 一、背景与目标
- 背景：开发一个辅助线下麻将的小程序
- 目标：为线下麻将玩家提供一个辅助工具，功能包括自定义/预置麻将规则并使用规则算分，开房间记账并自动关联记分规则
- 范围边界（不做什么）：

## 二、用户与场景
- 用户角色：线下麻将玩家
- 使用场景：
  - 玩家可以开房间记录线下麻将积分
  - 房主创建房间后通过“小程序分享”邀请其他玩家加入；不支持手动添加玩家与通过房间ID加入
  - 房间支持玩家手动输入每局积分或通过上传牌型图片进行 AI 记分（与所选规则关联）
  - 规则管理支持查看预置规则（川麻模版）、新建自定义规则（仅新建/删除）
- 成功标准（验收指标/容忍度）：

## 三、规则模版（AI 算分依据）
- 存储格式：Markdown（人类可读，AI 直接消费）
- 必需分节：
  - 桌面（`players`、`handSize`、`tiles.totalTiles/是否含风/箭/花`）
  - 庄家（`continuousDealer`、`dealerBonus.selfDrawAddBase/winAddBase`）
  - 结算（`payMode`：一人付/三家付、`basePoint`、`cap`）
  - 抓鸟（`enabled`、`count`、`hitBonus`）
  - 违例检查（查大叫/查花猪）
  - 番型（基础番、杠类、额外加成；示例：平胡、自摸、对对胡、清一色、明杠、暗杠、绝张）
- 预置规则：川麻·血战到底（已内置并可查看）
- 自定义规则：输入“名称 + 自然语言描述”，调用 LLM 渲染 Markdown；保存前执行规范校验（必需分节存在）；支持多轮交互补充信息完善规则

## 四、AI 服务与架构选择
- 上传图片并计算积分通过调用AI服务实现，目前使用火山引擎的API

## 五、记分计算程序
- 规则执行由程序根据规则配置文件（Markdown 对应的结构化字段）进行计算，确保兼容各类麻将规则（扩展番型、封顶策略、连庄与鸟规则等）。
- 图片解析由 AI 将胡牌情况转为规则执行输入（如手牌/副露/番型线索），与所选规则结合计算积分。

## 七、前后端接口（改造后）
- 前端仅持有最小状态，所有规则/房间数据通过后端接口管理。
- 规则：
  - `GET /api/rules` 返回 `{ id, name, preset, templateMarkdown, version }[]`
  - `POST /api/rules` 接收 `{ name, markdown | prompt }`；若为 `prompt`，后端调用 LLM 生成 Markdown 并返回
  - `DELETE /api/rules/:id` 删除（预置 `preset=true` 的需拒绝）
- 房间：
  - `POST /api/rooms` 创建并设置房主信息 `{ ownerUid, ownerName, ownerAvatar }`
  - `GET /api/rooms/:id` 返回详情（含玩家与轮次）
  - `PUT /api/rooms/:id` 保存轮次与规则关联
  - `POST /api/rooms/:id/join` 根据用户凭证加入房间
- AI：
  - `POST /api/ai/score` 上传图片代理火山引擎，返回 `{ score, detail }`
  - `POST /api/ai/rule-markdown` 渲染规则 Markdown，返回 `{ markdown }`

## 五、登录与加入
- 登录：首次进入必须微信头像/昵称授权（本地认证，不依赖后端）
- 加入：仅支持通过房间分享卡片进入后自动加入；不支持手动添加或输入房间ID加入

## 六、校验与错误处理
- 规则保存校验：
  - 结构完整性：玩家数/手牌数/番型数组存在
  - Markdown 分节：桌面/庄家/结算/抓鸟/违例检查/番型
- LLM 渲染失败回退：使用本地解析生成规范 Markdown

## 七、用户认证
- 使用微信小程序的账号认证，拉取用户昵称与头像用于房间标识与分享加入。
